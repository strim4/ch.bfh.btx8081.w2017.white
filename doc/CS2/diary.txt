
20.10.2017: Aufgabenstellung erhalten



Wir entwicklen eine Webapp, welche Menschen mit Depressionen unterstützen soll.

TASK1 von 20.10 - 02.11
____________________

In der ersten Projektphase sollen use cases und user requirements im Sinne von design thinking  evaluiert werden.
Diese Phase soll durch ein Interview, vorzugsweise mit einem Menschen mit Depressionen oder aber einer indirekt
betroffenen Person (Angehörige, Fachpersonal, usw.), durchgeführt werden.

Es können und sollen auch weitere Quellen, wie z.B. Fachliteratur verwendet werden.

Aus diesem Projektschritt sollen bereits erste Ideen für eine App entstehen. In Textform oder bereits mit
Scetches für Screens.

Die Ergebnisse werden am 02.11 in Form einer 10-minütigen Präsentation vorgestellt.

Detailbeschrieb zu Task1 siehe Dokument Aufgabenstellung_Task1.docx
Im Ordner Task01 auf dem GIT wurden alle Dokumente ausser die ausgefüllten Interviews gelöscht.

Präsentation am 02.11.2017 gehalten
Bericht und Präsentation auf dem GIT abgelegt, Task01 ist somit vollständig abgeschlossen.

TASK2 von 02.11 - 16.11.
____________________

In der zweiten Projektphase sollen ein UML-Domainmodell und zwei Sequenzdiagramme
a) Wenn der Benutzer bestimmte Informationen aktualisiert
b) Wenn das System den Benutzer über ein bestimmtes Ereignis informiert
entworfen werden.
Falls notwendig soll das UML-Klassenmodell, um die Attribute und Operationen,
welche für die Sequenzdiagramme benötigt werden, ergänzt werden.

Die Ergebnisse werden am 16.11.2017 in Form einer 10-minütigen Präsentation vorgestellt.
